# Build A First-Class Migration And Adoption Workflow

Created: 2026-04-11T14:49:34Z

## Description

Add a guided path for taking an existing repo with docs and notes into the Brain model without manual reconstruction. The first version can stay narrow as long as it preserves existing docs and produces a predictable local Brain layout.


## Acceptance Criteria

- [ ] A non-greenfield repo can be adopted through a documented CLI workflow
- [ ] The workflow preserves existing docs and does not clobber user content
- [ ] The resulting Brain layout is predictable and documented
