# Extract Verification Recipes From Repo And Successful Command Sources

Created: 2026-04-15T05:33:26Z

## Description

Derive reusable verification-recipe items from the repo and successful command history so compiled packets can include likely verification guidance with explicit provenance.


## Acceptance Criteria

- [ ] Verification recipes can be derived from observable sources such as CI config, make targets, package scripts, or bounded successful command history
- [ ] Recipe items preserve their source and strength so users can tell why a verification hint was included
- [ ] The first-wave extraction avoids turning Brain into a generic verification planner or a broad history miner
