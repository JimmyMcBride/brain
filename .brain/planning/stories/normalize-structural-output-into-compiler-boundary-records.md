# Normalize Structural Output Into Compiler Boundary Records

Created: 2026-04-15T05:33:26Z

## Description

Turn the existing structural repo derivation into normalized boundary records the compiler can query directly instead of relying on mostly human-oriented structural output.


## Acceptance Criteria

- [ ] Existing structural derivation is normalized into stable boundary records with IDs, labels, roles, root paths, and compiler-usable metadata
- [ ] Boundary record generation stays deterministic and refreshable from repo state
- [ ] The normalized model reuses existing structural work instead of creating a disconnected second structure system
