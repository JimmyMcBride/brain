# Brain Wrapper For Codex

When operating in a repo that includes `brain`, prefer shelling out to the CLI for note operations instead of editing vault files ad hoc.

- Use `brain search` for retrieval before broader grep-style scans of the vault.
- Use `brain add`, `brain edit`, `brain move`, `brain content ...`, and `brain undo` to preserve backups and history.
- Reindex deliberately after meaningful note changes.
